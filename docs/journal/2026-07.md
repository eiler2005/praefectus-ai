# July 2026

## 2026-07-05 — OpenClaw Gateway resource containment

Context:

- `maxtg_bridge` stopped delivering while the shared CX23 host was offline after a hard stop.
- The previous boot had repeated OpenClaw cgroup OOM/restart evidence and a high Gateway restart count.
- Existing CPU/RAM/PID caps were not enough because `openclaw-gateway` could keep restarting forever.

Changes:

- Keep OpenClaw Gateway at `0.90 CPU`, `1224m` RAM, no swap, and `256` pids.
- Add Gateway restart budget `on-failure:5`.
- Persist Docker json log caps for OpenClaw critical services at `10m x3`.
- Align LightRAG host limits with the live OpenClaw runbook: `2304m` RAM and `2816m` memory-swap.

Validation:

- Run `ansible-playbook --syntax-check playbooks/70-docker-limits-critical.yml`.
- Run `ansible-playbook playbooks/70-docker-limits-critical.yml --check --diff` before apply.
- After apply/recreate, verify Gateway with `docker inspect openclaw-openclaw-gateway-1` for memory,
  pids, restart policy, and logging config.

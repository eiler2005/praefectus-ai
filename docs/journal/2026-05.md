## 2026-05-09 — OpenClaw gateway memory limit +200m

**Что:** поднят host-level memory limit для `openclaw-openclaw-gateway-1` с 1024m до 1224m.

**Почему:** контейнер был healthy, но `RestartCount=90`; kernel journal показал cgroup OOM kills внутри Docker scope.

**Как:** через `70-docker-limits-critical.yml`: persistent `/opt/openclaw/docker-compose.override.local.yml` и immediate `docker update` без перезапуска.

**Результат:** лимит должен сохраняться при следующем `docker compose up`; после применения проверить `docker inspect` и `./verify.sh`.

## 2026-05-03 — Этапы 1–3: база + memory limits

**Что:** Инициализация vps_management репозитория + применение memory limits на все 13 контейнеров.

**Почему:** VPS показывал swap usage 1.6/2GB (признак memory pressure). OOM-killer может случайно убить любой процесс.

**Как:**
- Этап 1: структура, vault, disk-cleanup, verify, git init
- Этап 2: `60-docker-limits.yml` — 5 сервисов (signals-bridge 384m, telethon 256m, wiki-import 256m, agentmail-* 256m)
- Этап 3: `70-docker-limits-critical.yml` — openclaw-gateway 1224m (с 2026-05-09), omniroute 512m, lightrag 192m, redis 256m; ghostroute-console 450m через router_configuration zone

**Результат:**
- 11 из 13 контейнеров с лимитами (xray + xray-xhttp без лимита — стабильно <60MB)
- Сумма лимитов: 4424MB после повышения openclaw-gateway 2026-05-09 (overcommit ок, реальный worst case <3.5GB)
- Все лимиты применены немедленно через `docker update`, плюс override.local.yml для персистентности

**Что проверить:** `docker stats --no-stream` после рестарта любого сервиса.
